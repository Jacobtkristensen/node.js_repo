Git

Git is a version control system that allows developers to track changes in their code and collaborate with others, making it easy to see what changes have been made, and by who.
It is especially useful when working on a project with other developers, as it allows for easy merging of changes and makes it possible to resolve potential conflicts.
Reviewing each other's code is also a part of the process, which results in higher quality code, and ensures that it's not only you, who knows what your part of the project is responsible for.

Below are some of the most commonly used git commands 


      git init

Is used to instantiate a git repository in your project directory. This creates a .git folder in the root of the working directory, if one doesn't exist already. This command only needs to be run once per project. 


      git add -A

This command will stage ALL changes in the entire repository, whereas the   git add . will only stage files in the current directory and its subdirectories.

it is also possible to stage a single file by using git add [file]

git add -A is more convenient, especially when working with files in multiple directories. 


        git commit -m "your-message"

This command will commit the staged files to the repository. The -m flag is used to include a message with the commit. This is especially useful when looking back at previous commits to understand the changes that were made.

It is also good practice to describe whatever changes have been made, so that other developers have a clear understanding of the changes made. 


      git push origin main

This command will push the committed changes to the main branch of the remote repository. You can replace main with the name of the branch you're working on. 




    git status

Shows the current state of the working directory and staging area. It lists all the files that have been modified, are untracked, or are staged for commit.
This is good to use when you want to see an overview of the changes that have been made, and to see which files are staged, and which are not. 


    git checkout [branch-name]

Switches to the specified branch. When adding -b flag as in git checkout -b [branch-name], it will create a new branch and immediately switch to it. This command is used to navigate between different development branches and for creating new branches to work on. 


    git merge [branch-name]

Merges the specified branch into your current branch.
This allows you to combine the changes from another branch with your current branch. 


    git pull origin [branch-name]

pulls and merges changes from the remote branch to your local branch. This updates your current branch with the latest commits from the remote repository.
It's essential for keeping your local repository up-to-date with changes from others, or when working on different pc's.

Strings

Template literals are a way to concatenate strings in JS. They are denoted by backticks (`) and can contain variables using the ${} syntax. 


    const firstName = "Anders";
    const lastName = "Latif";
  
    const fullName = `My first name is ${firstName} and my last name is ${lastName}`
    console.log(fullName)
    // Result: My first name is Anders and my last name is Latif


Numbers

Here we try to add a string and a number together.
To combine the string "2023" and the number 1, we can use the parseInt() function to convert the string to a number before adding them together. 


    const year = "2023";
    const increment = 1;
    
    let newYear = parseInt(year) + increment;
      
    console.log(newYear);
    // Result: 2024


Here we try to add two strings together.
To combine the strings "1.10" and "2.30", we can use the parseFloat() function to convert the strings to numbers before adding them together. 


    const numberOne = "1.10";
    const numberTwo = "2.30";

    const sum = parseFloat(numberOne) + parseFloat(numberTwo);
    console.log(sum);
    // Result: 3.4


same as above, but we use the toFixed() method, to ensure that we get a number with two decimals.


    const anotherNumberOne = "1.10";
    const anotherNumberTwo = "2.30";
      
    let sum2 = parseFloat(anotherNumberOne) + parseFloat(anotherNumberTwo);
    sum2 = sum2.toFixed(2);      
    console.log(sum2);
    // Result: 3.40


Here we try to add three numbers together and then calculate the average.
We again use toFixed() to ensure that we get a number with five decimals. 


    const one = 10;
    const two = 45;
    const three = 98;

    let sum3 = (one + two + three) / 3;
    sum3 = sum3.toFixed(5);
    console.log(sum3);
    // Result: 51.00000

Type coercion

Type coercion in Javascript is the automatic or implicit conversion of values from one data type to another. Usually it's when you perform operations on values of different types, such as adding a string and a number together.
Understanding this concept is important, as it can lead to bugs and unexpected behavior.

Example of type coercion and strict equality operator in JavaScript 


    2 == "2" -> true 
    2 === "2" -> false //checks for both value and type
    '100' + 10 = '10010'

In the first example, the double equals operator coerces the string "2" to a number before comparing the two values, which results in true.
In the second example, the strict equality operator does not perform type coercion, so the comparison returns false.
In the third example, the plus operator concatenates the string "100" and the number 10, resulting in the string "10010".
